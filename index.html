Big O notation is a way to describe the performance or complexity of an algorithm. It describes the worst-case scenario for the amount of resources (such as time or space) that an algorithm will use as the size of its input grows. The most common types of time complexity are: O(1), which means the time it takes is constant, regardless of the size of the input. O(log n), which means the time it takes grows logarithmically with the size of the input. O(n), which means the time it takes grows linearly with the size of the input. O(n log n), which means the time it takes grows as the size of the input multiplied by the log of the size of the input. O(n^2), which means the time it takes grows as the size of the input squared. O(2^n), which means the time it takes grows exponentially with the size of the input. It is important to note that Big O notation only describes the worst-case scenario and does not take into account the specific inputs or characteristics of the algorithm. In addition, the Big O notation is used to describe the time complexity of an algorithm, but it can also be used to describe the space complexity, which is the amount of memory used by an algorithm. A good way to optimize the performance of an algorithm is to use a data structure or algorithm with a better time complexity. For example, using a hash table instead of a linked list can change the time complexity of a problem from O(n) to O(1) in the average case. When developing an algorithm, It is important to consider the time complexity and space complexity of the algorithm and try to optimize it as much as possible. This will make the algorithm more efficient and faster, which is particularly important for large inputs and real-world applications. In conclusion, Big O notation is a way to describe the performance or complexity of an algorithm and it's important to consider the time and space complexity when developing an algorithm and try to optimize it as much as possible.

function sum_char_codes(n: string): number {
    let sum = 0;
    for (let i = 0; i < n.length; ++i) {
        const charCode = n.charCodeAt(i);
        // Capital E
        if (charCode === 69) {
            return sum;
        }

        sum += charCode;
    }

    return sum;
}
